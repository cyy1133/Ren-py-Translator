# Phase 01 - Repository And Runtime Analysis

## 목표

- 현재 `Ren-py-Translator`의 실제 동작을 코드 단위로 파악한다.
- 어떤 지점이 테스트 게임 `Changiling tales`에 맞지 않는지 기록한다.
- 이후 리팩터링 기준선을 고정한다.

## 기존 상태 요약

- 원본 백엔드는 `game` 폴더 전체의 `.rpy`를 단순 스캔했다.
- 테스트 게임처럼 `game/*.rpy`와 `game/tl/ko/*.rpy`가 동시에 있는 경우, 원본과 번역 레이어가 섞여 분석된다.
- 번역은 파일 전체를 모델에 통째로 던지는 방식이었다.
- 성인 묘사 분리, 용어집, 세계관 규칙, 캐릭터 카드, 번역 블록 단위 검증이 없었다.
- `WebUI.HTML`은 일부 로컬 수정이 섞이며 문법이 깨진 상태였다.

## 이번 단계에서 고정한 판단

- 번역 레이어가 있는 게임은 `game/tl/<lang>`를 우선 분석한다.
- 모델은 텍스트만 번역하고, Ren'Py 형식 보존은 앱이 맡는다.
- 캐릭터 설정은 단순 `textarea` 하나가 아니라 `역할 / 톤 / 메모`로 분리한다.
- 성인 묘사는 번역 실패와 별개로 별도 큐를 유지한다.

## 코드 결과

- `RBackend.py`
  - 분석 API `/analyze_sources` 추가
  - 번역 API `/translate`를 블록 단위 파이프라인으로 교체
  - 게임 경로 / 업로드 파일 양쪽 지원
- `WebUI.HTML`
  - 새 작업대 레이아웃으로 교체
- `webui.js`, `webui.css`
  - 프론트 로직과 스타일 분리

## 아직 남은 것

- `.rpyc`만 있는 게임 자동 복구/디컴파일은 아직 미구현
- 실제 API 키를 넣은 번역 성공/실패 케이스는 별도 실사용 테스트 필요
