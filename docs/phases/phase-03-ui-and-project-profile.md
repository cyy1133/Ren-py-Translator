# Phase 03 - UI And Project Profile

## 목표

- 사용자에게 캐릭터별/세계관별 번역 설정을 직접 입력할 수 있는 UI를 제공한다.
- 설정을 JSON 프로필로 저장/복원한다.

## 현재 UI 구조

상단 제어 영역:

- 공급자
- OpenAI 인증 방식 (`API key` / `Codex OAuth`)
- API 키
- Codex CLI 명령 템플릿
- 모델명
- 대상 언어
- 배치 크기
- 호출 지연
- 성인 콘텐츠 메인 번역 포함 여부

분석 영역:

- 게임 `.exe` 경로 분석
- `EXE 선택` 후 즉시 분석
- `.rpy` / `.txt` 업로드 분석
- 분석 모드 라벨
- OpenAI OAuth CLI 상태 점검

탭 구조:

- `개요`: 대본 기반 자동 추정 결과 요약
- `대사/파일`: 대표 대사 미리보기, 파일 선택
- `용어집/세계관`: 세계관 메모, 보호 용어, 용어집 편집
- `캐릭터`: 좌측 목록 + 우측 상세 편집 작업대, 스프라이트 썸네일, 샘플 대사
- `성인 큐`: 메인 번역 제외 항목 확인
- `결과/로그`: 번역 실행, 결과, 로그
- 반응형: 좁은 폭에서 입력 버튼, 탭, 캐릭터 편집 영역 재배치

설정 영역:

- 세계관 설명
- 전반 톤 메모
- 형식/보존 규칙
- 보호/음차 용어 목록
- 용어집 테이블

캐릭터 영역:

- 분석 직후 화자별 기본 역할/말투/메모 자동 추정
- `biothumb_*` 또는 일반 스프라이트 정의에서 캐릭터 썸네일 자동 추출
- 좌측 목록에서 빠른 검색/선택
- 검색 입력은 실시간 필터링 중에도 포커스와 커서 위치 유지
- 캐릭터 클릭은 로스터 전체 재렌더를 피하고 우측 상세 패널만 갱신해 스크롤 위치 유지
- 우측 패널에 `AI 역할 추정 / AI 말투 추정 / 번역 메모` 요약
- 표시명
- 역할
- 말투/톤
- 세부 메모
- 샘플 대사

## 프로젝트 프로필 JSON

저장 항목:

- `provider`
- `model_name`
- `target_language`
- `batch_size`
- `api_delay`
- `include_adult_content`
- `game_path`
- `world_settings`
- `character_profiles`
- `selected_files`

의도:

- 테스트 게임별 프리셋을 재사용
- 캐릭터 말투와 용어집 누적
- 이후 추가 작업 페이즈에서 Notion/외부 DB로 옮기기 전 로컬 기준 포맷 유지
- 프로필 로드 시 게임 경로가 있으면 즉시 재분석해서 파일 선택/캐릭터 편집 상태를 UI에 다시 반영

## 남은 개선점

- 용어집 행 정렬/검색
- 선택 파일 필터링 UI
